# Jetpack Compose 课程网格界面开发实验报告

## 一、实验目的
1. 掌握 Jetpack Compose 常用基础布局组件使用，包括线性布局、盒子布局等。
2. 学习惰性网格组件的使用，实现双列网格列表与页面滚动效果。
3. 掌握自定义可复用UI组件的封装方法，提升代码整洁度与复用性。
4. 熟悉Android本地资源调用方式，完成图文组合页面搭建。
5. 理解Compose样式配置，包括圆角、间距、背景色、尺寸约束等界面美化操作。

## 二、实验环境
- 开发工具：Android Studio
- 开发语言：Kotlin
- UI开发框架：Jetpack Compose Material3
- 运行环境：Android 模拟器或安卓真机设备

## 三、实验原理
本实验基于Android最新声明式UI框架Jetpack Compose进行开发。
采用组件化开发思想，将页面拆分为顶部标题栏、网格列表区域、条目卡片组件。
利用惰性加载网格控件实现大量数据的高效展示，仅渲染屏幕可视区域内容，优化运行性能。
通过权重分配布局空间，实现顶部栏固定、内容区域自适应填充并支持滑动浏览。
结合Material3设计规范，完成卡片、图标、文字排版、色彩搭配等标准化界面设计。
采用静态数据源统一管理页面展示内容，实现数据与UI界面分离。

## 四、实验内容
1. 搭建整体页面结构，分为顶部标题栏与下方内容展示区。
2. 配置双列垂直网格布局，设置行列间距、内边距，规范页面边距。
3. 封装独立课程卡片组件，统一每个条目的布局结构与样式。
4. 完成卡片内部排版，左侧展示分类图片，右侧展示分类名称与课程数量信息。
5. 配置全局色彩风格，统一卡片背景、标题栏颜色、文字与图标色调。
6. 绑定本地静态数据源，循环渲染所有课程分类条目。
7. 实现页面纵向滚动功能，完整展示全部分类内容。

## 五、实验步骤
1. 新建Compose安卓项目，配置好项目基础依赖与主题文件。
2. 创建实体数据模型类，定义分类名称、数量、图片资源对应字段。
3. 编写静态数据源工具类，批量初始化所有课程分类数据。
4. 在主活动中编写整体页面布局，完成顶部标题栏样式编写。
5. 引入网格布局控件，设置固定两列布局参数与布局权重。
6. 单独编写卡片组合函数，完成图片、文本、图标组合排版。
7. 绑定数据源，遍历集合渲染每一个课程卡片。
8. 调试界面尺寸、圆角、间距、配色，保证页面显示整齐美观。
9. 运行项目，测试滚动效果、条目排版、资源加载是否正常。

## 六、运行结果
程序运行后，APP首页顶部显示纯色固定标题栏，下方以双列卡片形式展示所有课程分类。
每个卡片布局统一，左侧为分类配图，右侧显示分类名称与对应课程数量。
页面支持上下自由滑动，能够完整浏览全部十六个分类条目。
整体界面风格统一，圆角卡片、合理间距、柔和配色，界面简洁美观，交互流畅无卡顿。

## 七、实验总结
通过本次实验，熟练掌握了Jetpack Compose布局开发核心知识点，理解了声明式UI的开发逻辑。
学会了网格列表的使用与自适应布局适配，掌握了自定义组件封装的开发思路。
熟悉了安卓图片、文字、图标等本地资源的调用方式，能够独立完成基础图文页面开发。
同时了解了惰性加载组件的性能优势，为后续复杂列表开发打下基础。
在样式调整过程中，提升了UI排版与审美调整能力，能够按照设计要求还原页面效果。
后续可继续拓展功能，例如添加条目点击事件、分类筛选、数据动态更新等功能。
